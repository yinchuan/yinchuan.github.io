A brief record of blog setup

Published: Wed 08 March 2023
By yinchuan

In Computing.

A beief record for future me.

This blog is a static site generated by Pelican and is hosted on GitHub User Page. The articles are written in markdown and built to html by Pelican. The source markdown files are git versioned and pushed to a GitHub repo. Let's call it source_repo. The output html files are pushed to another repo. Let's call it publish_repo.

Basic process

The workflow of publishing content is:

  1. load Python virtualenv for Pelican
  2. edit articles in content directory
  3. start Pelican local server to review the built site before publishing
  4. visit the local sever in a web browser
  5. commit changes in content to master branch
  6. push master to source_repo
  7. run Pelican to build sources files to output htmls
  8. push output files to publish_repo

An improved process

Most steps can be automated using shell alias and git hook. The improved workflow is:

  1. run pelican_on. This is a shell alias which does:
    1. load Python virtualenv
    2. cd to Pelican source directory
    3. start Pelican local server. This needs to be run in Pelican source directory.
    4. start a new shell and cd to content directory in Pelican source. This is where the articles are. Starting a new shell is to use Vim for writing articels.
    5. open local server homepage in a web browser.
  2. write articles and review in browser.
  3. Commiting articles. This will trigger a git post-commit hook which does:
    1. push master to source-repo
    2. run gh-import to build output to gh-pages branch.
    3. push gh-pages branch to master branch of publish-repo.
