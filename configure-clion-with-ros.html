Configure-CLion-with-ROS

Published: Tue 07 March 2023
By yinchuan

In Computing.

I'm following the ROS Tutorial: Writing a Simple Publisher and Subscriber (C++). I've finished the tutorials before this one so I've finished setting up dependencies for message and service. I'm going to use CLion for C++ developing. Following the official documentation and a little tweaking, I managed to configure CLion with ROS.

The results:

  • developing ROS node in CLion
  • debugging a node as normal C++ application
  • don't clash with catkin_make

Steps:

  1. Lauch CLion in a sourced environment. As the offcial documentation said, the easiest way is to run CLion in a sourced shell.
  2. open <WORKSPACE_DIRECTORY>/src as a project in CLion.

  3. Change settings in Build, Execution, Deployment|CMake

    1. In Build Directory, choose: <WORKSPACE_DIRECTORY>/build
    2. In CMake options, add: -DCATKIN_DEVEL_PREFIX:PATH=<WORKSPACE_DIRECTORY>/devel
    3. (Optional) In Generator, choose: Unix Makefiles. This is what catkin_make uses while CLion(2022.2) uses Ninja by default. If you don't change this, you need to delete <WORKSPACE_DIRECTORY>/build before using catkin_make.
    4. in <PACKAGE>/CMakeLists.txt, uncommentting this line to use c++11 otherwise the code in the tutorial won't be built.

    add_compile_options(-std=c++11)
